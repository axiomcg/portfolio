# Bybit Trading Bot

Событийно-управляемый алготрейдинг-бот для **бессрочных USDT-фьючерсов Bybit** (linear
perpetual). Слушает Telegram-канал с сигналами о резком росте монет и на каждый сигнал
максимально быстро открывает **короткую позицию** по соответствующему фьючерсу — ставка
на откат после всплеска (mean-reversion / «фейд пампа»). Защита — серверный стоп-лосс на
бирже; полный леджер сделок в PostgreSQL; управление через Telegram.

## Как работает стратегия

- **Источник сигнала — внешний Telegram-канал-сигналка.** Хендлер ловит посты канала, триггер
  входа — эмодзи `🟢` + матч регуляркой тикера и процента роста (`#TICKER: +X.XX%`). Тикеры из
  стоп-листа пропускаются; повторный вход по тому же символу блокируется, пока позиция в работе
  (гард по членству символа в `ORDER_INFO`, см. risk-менеджмент).
- **Направление — short-only:** монета резко выросла → бот шортит, ожидая разворота. Это
  контртренд-логика «фейд пампа».
- **Вход** — рыночный ордер, объём `qty = POSITION_USD / price`, округлённый до биржевого шага
  лота (`qtyStep` из instrument info, кэшируется).
- **Выход** — серверный стоп-лосс, прикреплённый атомарно к входному ордеру (нет окна «позиция
  без стопа»), либо ручное закрытие командой из Telegram. Причина закрытия (user / stop-loss /
  ликвидация) фиксируется в БД.

## Risk management

- **Серверный (биржевой) стоп-лосс** в момент открытия (дефолт 0.8%).
- **Контроль плеча** `ensure_leverage_set`: кэш с TTL 600с, до 3 ретраев при IP-rate-limit,
  обработка специфичных кодов Bybit (`110043` — «leverage not modified»), фолбэк на чтение
  фактического плеча из API (дефолт 12.5x).
- **Защита от дублей/пирамидинга** — главный гард: символ уже в `ORDER_INFO` (in-memory) → вход
  блокируется; плюс стоп-лист тикеров. (Кэш `PROCESSED_SIGNALS` ведётся только для учёта/логов,
  как дедуп-гард он не используется.)
- Фиксированный номинал позиции (`POSITION_USD`), обработка ликвидаций на уровне учёта.

## Архитектура

- **Асинхронность + многопоточность:** главный поток — `asyncio` на **uvloop**, `gather` двух
  WS-задач; Telegram-бот (aiogram) — в отдельном демон-потоке со своим event loop.
- **Два WebSocket-стрима Bybit v5:**
  - **Приватный** (`order`/`execution`/`position`) с ручной **HMAC-SHA256** аутентификацией —
    источник истины об исполнениях и состоянии позиций.
  - **Публичный** (tickers) — отдельный поток рыночных цен (вспомогательный кэш `PRICE_CACHE`).
    В текущем коде в путь принятия решения НЕ заведён: цена входа берётся REST-ом, а подписка
    захардкожена на `tickers.BTCUSDT`.
- **REST через `pybit`** (Bybit Unified Trading v5): ордера, плечо, instrument info, позиции,
  отмена ордеров. Сериализация WS — `orjson` (скорость).
- **Леджер сделок (самая зрелая часть):** PostgreSQL 15 + **SQLAlchemy 2.0** (typed
  `Mapped`/`mapped_column`) + **Alembic** (3 миграции). Таблица `trade_log` с **FSM состояний
  сделки** (`Undefined / Opened / ClosedByUser / ClosedByStopLoss / ClosedByLiq`), агрегацией
  частичных филлов, взвешенной ценой входа, трекингом комиссий и **расчётом PnL через
  hybrid-property** на уровне ORM.
- **Контейнеризация:** Docker (python-alpine) + docker-compose (бот + Postgres), авто-`alembic
  upgrade head` при старте, поддержка нескольких инстансов через `app_id`.
- **Отказоустойчивость:** авто-reconnect обоих WS с фиксированной задержкой (приватный — 1с,
  публичный — 5с; не экспоненциальный бэкофф), graceful shutdown по SIGINT/SIGTERM.

## Низкая латентность

Путь «сигнал → ордер» инструментирован замером времени в мс; получение цены и установка плеча
вынесены в отдельные `asyncio`-таски (намерение — параллелизм), объём ордера считается заранее,
вход — рыночный ордер. Целевой сценарий — sub-second реакция на сигнал. **Оговорка:** REST идёт
через синхронный `pybit`, поэтому на happy-path «параллельные» подзадачи выполняются фактически
последовательно и блокируют event loop; подтверждённого трек-рекорда латентности нет (см. ниже).

## Telegram

Двойная роль: **источник сигналов** (парсинг канала) и **рантайм-панель управления**
(вкл/выкл торговли, изменение размера позиции и стопа на лету, ручное закрытие, статус).

## Стек

Python 3.11, asyncio + uvloop, `pybit` (REST), `websockets` (WS), `aiogram` (Telegram),
`orjson`, SQLAlchemy 2.0 + Alembic + psycopg2, PostgreSQL 15, Docker + docker-compose.
