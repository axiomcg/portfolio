# Intake — медиа для галерей проектов

Под каждый проект есть своя папка. **Кидай скрины и видео в папку нужного проекта** —
я (Claude) прогоню их через оптимизацию (WebP для картинок, web-MP4 + постер для видео),
сложу в `assets/` и допишу `assets/manifest.json`. Код трогать не нужно — лента каждого
проекта строится из манифеста по полю `project`.

## Папки
```
intake/hookora/        → лента проекта Hookora
intake/whispo/         → лента проекта Whispo
intake/hh-agent/       → лента проекта hh.ru Job Agent
intake/mband/          → лента проекта MBand
intake/vikishop/       → лента проекта VikiShop
intake/telegram-bots/  → лента проекта Telegram-боты (сейчас ленты нет — добавлю, как появятся файлы)
```

## Что класть
- **Скрины**: `.png` / `.jpg` (любой размер — пережму, широкие > 1600px ужму по ширине).
- **Видео-воркфлоу** (зацикленные, автозапуск при пролистывании): `.mov` / `.mp4` / `.webm`.
  - Горизонталь или вертикаль — без разницы (видео в ленте показывается целиком).
  - Звук не нужен (в ленте всегда muted + loop). 5–20 сек идеально.

## Имя файла = подпись (опционально)
Назови файл по-человечески — я возьму это как подпись карточки.
Можно с порядковым префиксом, чтобы задать порядок в ленте:
```
01-brain-graph.png        → «Brain graph»
02-pgvector-search.png    → «Pgvector search»
03-dictation-demo.mov     → видео «Dictation demo»
```
Хочешь точную подпись на двух языках — напиши её рядом в сообщении, проставлю EN/RU.

## Как сдать
Накидал файлы по папкам → напиши «готово, медиа в intake».
Я оптимизирую, обновлю манифест и покажу обновлённые ленты.
